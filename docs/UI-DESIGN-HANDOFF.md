# Orosaga / YiShanOS UI 设计交接规范

> 文档版本：1.0  
> 基准项目：Orosaga（移山科技内部知识站）  
> 目标用途：让另一套 YiShanOS 项目延续相同的视觉语言、素材板式与交互气质，而不是逐页照搬现有页面。

## 1. 设计定位

Orosaga 的核心气质是“安静、清晰、可长期使用的山中工作地图”。它不是营销落地页，也不是传统后台模板，而是把知识、组织、流程和成长路径组织成一套偏编辑式的工作界面。

设计关键词：

- 温润纸张：暖白底色、极轻纹理、低对比边界。
- 山水隐喻：山脊、雾带、路径、等高线、海拔与营地用于表达层级和进程。
- 编辑感：中文衬线标题配紧凑无衬线正文，信息层级明确。
- 工作导向：内容可扫读、入口明确、交互克制，不做装饰性大卡片堆叠。
- 独立页面：总览负责导航，复杂模块进入独立页面，不把所有内容塞进单页首页。
- 轻动效：只解释状态和方向，不用持续炫技动画干扰阅读。

## 2. 继承与适配边界

| 应保持一致 | 在 YiShanOS 中应适配 |
| --- | --- |
| 暖纸底、深山绿、陶土橙的主色关系 | 产品名称、Logo、页面标题和内容术语 |
| 衬线展示字 + 无衬线正文字体组合 | 首页主叙事，可从“知识地图”改成“工作台 / 系统地图” |
| 72px 桌面顶栏、1180px 内容宽度、紧凑圆角 | 导航栏目数量、业务模块顺序和权限入口 |
| 薄边框、少阴影、无嵌套卡片的布局原则 | 卡片语义色与模块图标，不要求一一对应 |
| 山路、营地、节点等进程隐喻 | 数据密度；运营系统可比知识站更紧凑 |
| Lucide 图标、清楚的 hover / focus / active 状态 | 人员照片、业务数据、外链和文案 |
| 980px / 680px 两级响应式策略 | 移动端功能优先级和隐藏项 |

不要直接把 Orosaga 字样、`山海经`副标题、同事头像或现有页面业务文案复制到 YiShanOS。应复制视觉规则和交互逻辑，再替换产品内容。

## 3. 当前技术基线

- React + TypeScript + Vite
- 图标：`lucide-react`
- 样式：全局 CSS，集中在 `src/styles.css`
- 当前路由：在 `src/App.tsx` 中依据 `window.location.pathname` 手动分发
- 当前页面：`/`、`/company`、`/workflow`、`/organization`、`/voices`
- 字体来源：Google Fonts 在线加载

YiShanOS 如果使用 React Router、Next.js 或其他路由方案，应保留独立页面结构，但不必沿用手动路由实现。

## 5. 字体与信息层级

### 5.1 字体角色

- 中文正文与 UI：`Noto Sans SC`，400 / 500 / 600 / 700。
- 中文标题：`Noto Serif SC`，600 / 700。
- 英文品牌、数字与章节编号：`DM Serif Display`，400。
- 不使用负字距；当前项目所有标题和标签都保持 `letter-spacing: 0`。

### 5.2 推荐字号

| 层级 | 桌面 | 移动 | 规则 |
| --- | --- | --- | --- |
| 首页主标题 | `72–138px` | 约 `56–72px` | 只用于真正的首页 Hero |
| 场景页 Hero | `44–68px` | `40px` | 如营地地图 |
| 详情页 H1 | `44–48px` | `34–38px` | 公司、流程、组织页 |
| 区块 H2 | `30–42px` | `24–30px` | 中文衬线 |
| 卡片标题 | `15–25px` | `14–22px` | 随容器密度调整 |
| 正文 | `12–14px` | `12–14px` | 行高 `1.7–1.9` |
| 标签 / eyebrow | `9–11px` | `9–11px` | 600–800，短句 |

Eyebrow 统一使用“英文短语 · 中文栏目”的格式，例如：

```text
Operating workflow · 运营工作流
People & collaboration · 组织与协作
Field notes · 同事分享
```

## 6. 布局、间距与形状

### 6.1 页面骨架

- 标准内容宽度：`1180px`。
- 宽场景上限：营地路线可放宽到 `1440px`。
- 桌面水平 gutter：`28px`；移动端：`18px`。
- 顶栏：桌面 `72px`，移动 `64px`，固定在顶部。
- 大区块垂直留白：桌面通常 `96–128px`；移动通常 `64–86px`。
- 详情页 Hero 顶部需补固定顶栏空间，常用 `138px 0 64px`。
- 复杂文章页可用 `210px + 剩余宽度` 的粘性目录布局。

### 6.2 间距节奏

优先使用以下间距序列：`4 / 8 / 12 / 16 / 20 / 24 / 28 / 34 / 42 / 54 / 72 / 96 / 120px`。

- 控件内部：8–16px。
- 同组元素：8–14px。
- 卡片内部：20–30px。
- 模块间：34–54px。
- 页面大区块间：72–120px。

### 6.3 圆角与阴影

- 标签：`4px`。
- 输入框、按钮、工具条：`6–7px`。
- 卡片、弹窗、Logo 容器：最多 `8px`。
- 头像、节点、方向按钮可以使用圆形。
- 禁止把所有容器都做成大圆角胶囊。
- 普通卡片默认无明显阴影，只在 hover 时使用 `0 18px 40px rgba(30,48,41,.1)`。
- 模态浮层可使用更深阴影，如 `0 30px 80px rgba(9,22,17,.32)`。

## 7. 通用组件配方

### 7.1 顶栏

- 固定定位，三列布局：品牌 / 当前页面或导航 / 操作区。
- 背景：`rgba(250,251,248,.82–.95)`。
- `backdrop-filter: blur(20px) saturate(150%)`。
- 底边只用 1px 低对比边线。
- 桌面高度 72px；移动端 64px。
- 详情页右侧使用“返回知识地图”；首页右侧为搜索和菜单。

### 7.2 品牌锁定组

- Logo：桌面 `38 × 38px`，移动 `34 × 34px`，圆角 `8px`。
- 英文品牌：DM Serif Display，20px / 18px。
- 中文副标：无衬线 12px、600，使用 muted 色。
- YiShanOS 应替换品牌文字与 Logo，但保持横向尺度关系。

### 7.3 按钮

- 主按钮：高度 `48px`，左右内边距 `20px`，圆角 `7px`。
- 紧凑按钮：高度 `42px`，左右内边距 `16px`。
- 图标按钮：`36 × 36px`；关键路径圆形按钮 `40 × 40px`。
- 主命令可使用深墨色或深绿实底；图标放在文字右侧表示前进。
- hover：上移 1–2px 或图标右移 3–4px，不同时叠加大缩放。
- 禁用态必须降低对比、去除阴影，并取消跳转能力。

### 7.4 卡片

- 卡片只用于独立、可重复的信息单元，不把整段页面 section 包成漂浮卡片。
- 默认 1px 低对比边框，6–8px 圆角。
- 知识入口卡可使用浅语义色，大卡允许跨两列。
- 卡片图标放在 42px 圆形浅色容器内，右上角可放 34px 方向图标。
- 避免卡片里再套卡片；内部层级用分隔线、留白和底色带表达。

### 7.5 标签、状态与数字

- 标签尺寸 9–11px，圆角 4px，边框比文字更浅。
- 编号、海拔和统计数字优先使用 `DM Serif Display`。
- 当前状态用深绿实底或陶土橙 3px 底线，不依赖颜色时需同时改变形状或文字。

### 7.6 模态与纵向名片

- 遮罩：`rgba(12,18,16,.42)` + `blur(8px)`。
- 桌面名片为窄而纵向的内容容器，最大圆角 8px。
- 上半区可用深绿承载头像、姓名和部门；下半区使用白底展示介绍、协作关键词和动作。
- 关闭按钮固定在右上角，必须有 `aria-label`，支持 Esc 与点击遮罩关闭。
- 移动端名片从底部对齐，最大高度 `calc(100svh - 24px)`，内部滚动。

## 8. 页面模板

### 8.1 总览 / 知识地图首页

结构顺序：

1. 固定顶栏。
2. 全首屏山水 Hero，品牌或产品名是第一视觉信号。
3. 右侧可互动的今日信息、状态或成长路径。
4. 模块入口网格。
5. 深色成长路径带。
6. 最新内容列表。
7. 共建入口与深色页脚。

YiShanOS 首页应作为系统地图和任务入口，不要变成产品宣传页。Hero 下方要露出下一段内容的提示，避免首屏完全封闭。

### 8.2 编辑式内容页

适合公司介绍、制度、方法论、帮助中心。

- 固定顶栏。
- 桌面使用粘性左目录 + 右侧文章。
- 文章内容本身为连续白色平面，以 section 分隔线划分章节。
- 平板将左目录变成顶部横向滚动目录。
- 移动端缩小内边距，复杂对照图转为纵向排列。

### 8.3 运营工具页

适合工作流、任务阶段、标准操作程序。

- 紧凑 Hero 说明目标与使用原则。
- 深色摘要条展示关键统计。
- 6 步以内用横向阶段 Tab；平板 3 列，手机 2 列。
- 内容面板左侧深色“阶段定义”，右侧白色“检查项与动作”。
- active 状态同时使用白底、深绿图标与陶土橙底线。

### 8.4 组织与人员页

适合团队、角色和权限关系。

- Hero 下方直接放搜索与部门筛选，不另套装饰卡。
- 顶层节点居中并平行摆放关键人员。
- 部门面板用低饱和蓝 / 绿 / 紫 / 玫瑰区分。
- 人员按钮固定高度，头像、姓名、角色、HEAD 标签和外跳图标保持稳定列宽。
- 点击人员打开纵向名片，不在组织图内展开详情。
- 移动端去掉复杂连接线，部门改为单列，保证阅读优先。

### 8.5 路线 / 营地场景页

适合知识贡献、成长等级、项目阶段和内容分层。

- 深绿地形 Hero，陶土色太阳作为唯一高饱和视觉锚点。
- 桌面左侧为 sticky 路线轨道，右侧为按海拔分段的长页面。
- 每段至少接近一屏高度，让滚动本身表达“上山”。
- 营地节点以文档数量或贡献值分层，点击跳转到详情或外链。
- 移动端隐藏 sticky 轨道，将绝对定位营地转换为稳定的 2 × 2 网格。

## 9. 山水素材板式

项目的主要山水场景不是位图，而是 CSS 图形层。这让素材可响应、可换色，也便于 YiShanOS 复用。

### 9.1 首页山水层

由下列层级组成：

```text
Hero 内容
薄雾带（模糊的半透明白色椭圆）
近景山脊（深绿）
中景山脊（中绿）
远景山脊（浅绿）
纸面太阳（低透明陶土粉）
暖白到浅青的背景场
```

山脊使用 `clip-path: polygon(...)`，每一层高度、透明度和轮廓不同。动画只做 5px 左右的呼吸位移与约 1% 横向缩放；雾带缓慢横移 40px 左右。

复用规则：

- 保持 3 层山脊，避免只做一块扁平剪影。
- 近景最深、最矮；远景最浅、最高。
- 轮廓不要完全对称，也不要出现尖锐重复锯齿。
- 太阳只出现一次，使用陶土橙 / 粉，不使用发光渐变球。
- 山景服务于信息层级，不能遮挡按钮、标题和正文。

### 9.2 营地地形层

- Hero 山体使用 2 层 polygon 山峰，近景 `#1e332b`，远景 `#39564b`。
- 等高线使用 3 个大椭圆边框，从页面右下方溢出。
- 路径使用 1px 虚线，略微旋转；首尾用空心圆锚点。
- 营地 pin 为 42px 圆形，深绿与陶土色交替。
- 营地卡片使用近白半透明背景、小阴影和 6px 圆角。

### 9.3 何时使用真实图片

- 人员、真实项目、产品界面或客户案例需要可核验细节时，使用真实图片。
- 抽象流程、知识层级、成长阶段使用 CSS 地形和 Lucide 图标。
- 不要用模糊库存风景图代替真实内容，也不要用装饰图片压低文字可读性。

## 10. 素材清单与使用规则

| 素材 | 当前路径 | 规格 / 说明 | YiShanOS 处理 |
| --- | --- | --- | --- |
| 品牌图标 | `public/favicon.svg` | 64 × 64 viewBox；纸白、陶土太阳、双层山水 | 建议复制构图逻辑并替换品牌标识 |
| 人员头像 | `public/team/*.png` | 256 × 256，RGB PNG，共 31 张 | 仅当前组织数据使用，不作为通用设计资产 |
| 首页山水 | `src/styles.css` 中 `.shanshui-scene` 等 | CSS 生成，无独立图片 | 可复用结构与色彩关系 |
| 营地地形 | `src/styles.css` 中 `.camps-hero-terrain` 等 | CSS 生成 | 可用于贡献、阶段、等级地图 |
| 图标 | `lucide-react` | 统一线性图标，常用 14–24px | YiShanOS 继续使用同一图标库 |
| 噪点纸纹 | `body::before` 内联 SVG turbulence | 固定全屏，透明度 `0.025` | 可复用，不能提高到影响文字清晰度 |

人员头像的标准处理：

- 源图保持正方形，建议至少 256 × 256。
- UI 中使用 `object-fit: cover` 与圆形裁切。
- 头像缺失时显示 2 个以内的中文字符，不使用破图占位。
- 图片 `alt` 使用“姓名 + 头像”；纯装饰 Logo 可使用空 `alt`。

## 11. 交互与动效

### 11.1 时间与曲线

- hover / focus：`160–180ms ease`。
- 卡片位移与边框变化：`200ms ease`。
- 内容面板切换：`200–220ms ease`。
- 模态出现：遮罩 150ms，容器 180ms `cubic-bezier(.2,.8,.2,1)`。
- 长时间环境动画：山脊 9–13s，雾带 16–20s，必须非常轻。

### 11.2 状态行为

- 链接前进：箭头右移 3–4px。
- 卡片可点击：整体上移不超过 5px，并加深边框。
- 路线节点当前态：圆点变为深绿实底并轻微放大。
- Tab 切换：内容淡入并上移 4px，不做横向大滑动。
- 外链必须用外跳图标，并加 `target="_blank" rel="noopener noreferrer"`。

### 11.3 减少动态

必须保留：

```css
@media (prefers-reduced-motion: reduce) {
  html { scroll-behavior: auto; }
  *, *::before, *::after {
    animation-duration: .01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: .01ms !important;
  }
}
```

## 12. 响应式规范

### 12.1 断点

- `> 980px`：桌面布局。
- `681–980px`：平板 / 小桌面布局。
- `<= 680px`：手机布局。
- 最小支持宽度：`320px`。

### 12.2 平板策略（980px）

- 顶部主导航收进菜单。
- 4 列卡片变 2 列。
- 双栏 Hero 在空间不足时转单栏。
- 组织架构 4 部门转 2 × 2。
- 工作流阶段从 6 列转 3 × 2。
- 文章左目录变为顶部横向 sticky 导航。
- 营地 sticky 轨道从 190px 收窄到 150px。

### 12.3 手机策略（680px）

- 顶栏高度 64px，隐藏中间页面位置文字。
- 搜索只保留图标，保持 36 × 36px 点击区域。
- 主体 gutter 18px，标题不能按视口宽度无限缩放。
- 所有复杂双栏变单栏；组织连接线直接隐藏。
- 工作流阶段 2 列；工作流详情上下排列。
- 人员名片底部对齐，最大高度受 `100svh` 限制。
- 营地使用 2 × 2 网格，取消节点绝对定位。
- 横向筛选和文章目录允许滚动，不压缩到文字换行失控。

## 13. 无障碍与内容规则

- 所有可点击对象必须使用 `a` 或 `button`，不使用无语义 `div` 代替。
- 焦点统一为 `2px solid #2167eb`，外偏移 3px。
- 图标按钮必须有 `aria-label`；Tab 使用 `role="tablist"` / `role="tab"` / `aria-selected`。
- 弹窗使用 `role="dialog"`、`aria-modal="true"`，支持 Esc 关闭。
- 不能只靠颜色表达 active、HEAD、成功或禁用状态。
- 正文优先 12–14px 配 `1.7–1.9` 行高；低于 10px 的文字仅用于次要元数据。
- 按钮和卡片内的最长中文标题必须允许换行；英文标识使用 `overflow-wrap: anywhere`。
- 文案保持短、具体、可行动。页面里不要加入解释 UI 怎么用的宣传式段落。

## 14. YiShanOS 落地顺序

1. 先迁移字体、颜色、宽度、圆角、阴影、动效令牌。
2. 建立 `AppShell`、`Topbar`、`BrandLockup`、`PageFooter` 和 `SectionWrap`。
3. 建立按钮、图标按钮、标签、输入框、Tab、Modal 五类基础控件。
4. 根据 YiShanOS 信息架构建立独立路由，不把所有模块继续堆在首页锚点下。
5. 首页先搭系统地图和核心入口，再加入一处山水场景，不要每个 section 都使用山景。
6. 运营型页面复用“摘要条 + 阶段 Tab + 双区详情面板”。
7. 人员 / 权限页复用“筛选 + 层级图 + 纵向名片”。
8. 最后加入纸纹、山脊呼吸、hover 位移等质感层。

建议将现有全局 CSS 拆为：

```text
styles/
  tokens.css
  reset.css
  shell.css
  components.css
  scenes.css
  responsive.css
```

这不是强制要求，但 YiShanOS 页面继续增加时，比单个超长 `styles.css` 更便于维护。

## 15. Do / Don't

### Do

- 用连续页面平面和分隔线组织信息。
- 用一处明确的山水场景建立品牌识别。
- 用低饱和色区分业务语义。
- 把复杂模块放到独立 URL。
- 为桌面、平板、手机分别设计布局变化。
- 保持图标尺寸、控件高度和卡片网格稳定。

### Don't

- 不做营销式大圆角 Hero 卡片。
- 不在卡片里继续套卡片。
- 不使用紫蓝渐变、发光色球或大量玻璃拟态作为主视觉。
- 不让每个模块都变成漂浮白卡。
- 不用超大标题填满业务工具页面。
- 不用持续缩放、弹跳或大幅视差干扰工作。
- 不直接复用当前员工照片和内部业务文案到新项目。

## 18. 已知工程注意事项

1. 当前项目使用 `window.location.pathname` 手动分发页面。YiShanOS 若部署到静态服务器，必须配置 SPA fallback，否则直接访问 `/company`、`/workflow` 等地址可能返回 404。
2. 字体目前从 Google Fonts 在线加载。在企业内网或访问受限环境中，应改为本地托管字体文件，避免页面退回系统字体后视觉变化过大。
3. 当前 `src/styles.css` 已承载多页面样式。YiShanOS 迁移时应先抽令牌与共享组件，再按页面域拆分，避免继续复制选择器。
4. CSS 山水可以复用构造方式，但应调整轮廓和内容位置，形成 YiShanOS 自己的场景，而不是生成完全相同的首页截图。
