- 全新 PostgreSQL 验证库可从零执行 migration 和 seed，19 张业务表及字段 SQL 注释缺失数为 0。
- API smoke 覆盖匿名拒绝、三角色 200/403、CSRF、统一校验错误、立即发布、409、知识源版本、搜索、退出与会话撤销。
- 34 项 Playwright 稳定通过，包含 6 个页面在 1440、1024、768、390、320px 的 30 份视觉基线、真实 404、员工弹窗键盘关闭、后台立即发布和首页严重/致命无障碍扫描。
- lint、Prettier、TypeScript、单元/契约覆盖率、production build、bundle budget、敏感 bundle、仓库 secret、许可证和 npm 官方漏洞审计均通过；npm audit 为 0 vulnerability。

## 正式放量前仍需关闭的工程闸门

- 服务器尚未启动 Orosaga 容器、`orosaga` schema 仍为空；RDS 内网免 SSL 预检已经验证 PostgreSQL 16、私网解析、`pg_stat_ssl=false`、schema 权限与 `pg_trgm`，迁移仍必须等待手工快照成功。
- 当前覆盖率闸门证明已纳入测试的核心模块达到阈值，不等同于整个仓库所有源文件达到 80%；应在上线前扩大采集范围并关闭剩余分支缺口。
- 后台已覆盖页面发布、历史回滚、知识源和同步；工作流、系统入口、员工扩展资料目前具备受保护 API，完整专用表单和受限 block editor 仍需补齐。
- 组织与 Wiki 已实现完整拉取后单事务切换和同会话 advisory lock，仍需真实租户验证分页、姓名唯一映射、权限失败及旧口径对账；飞书头像增量写入 OSS 不属于首次31张迁移脚本。
- 99.9% 可用性、API/搜索 p95、RPO/RTO 和连续运行指标只能在 staging/production 通过监控与演练确认。
