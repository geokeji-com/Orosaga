# 生产运行手册

## 健康与告警

- `/healthz` 是唯一匿名健康端点；`/readyz` 会验证数据库并要求内部会话或受控运维访问。
- 监控登录成功率、403/5xx、API p95、搜索 p95、连接池、同步新鲜度与失败率。
- Wiki/组织连续 24 小时无成功运行、飞书 401/403、备份失败或 5xx 超阈值立即告警。

## 备份

RDS 开启 PITR 与每日快照并保留 30 天；OSS 开启私有 ACL、版本控制与误删保护。每季度在隔离环境恢复数据库和 OSS，并记录 RPO/RTO。目标：用户编辑数据 RPO 1 小时、RTO 4 小时。

## 发布

GitHub Actions 对同一 SHA 执行 lint、格式、类型、单元/契约测试、构建、bundle 敏感信息与预算扫描、依赖审计、许可证、SBOM 和 CodeQL。staging smoke 通过并人工批准后，提升同一 Web artifact 与 API/Worker 镜像。

数据库只采用 expand-contract：先增加兼容字段/表，部署双读写或回填，最后在后续版本移除旧结构。禁止把逆向删除脚本当作应用回滚手段。

## 日志规则

允许：request ID、HTTP 状态、路由模板、耗时、非敏感资源 ID、同步计数。禁止：飞书 token、Cookie、CSRF token、正文、手机号、邮箱、完整个人资料和 KMS 返回值。

## 外部控制面待办

仓库不包含阿里云和飞书凭据，因此 ECS、ALB、RDS、OSS、KMS、SLS、TLS、DNS 与飞书应用发布必须由有权限的管理员执行。完成后将实际资源 ID、负责人和恢复演练证据登记到内部运维系统，不写入公开构建产物。
