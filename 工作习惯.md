# 用户工作习惯

- 用户倾向用简短指令授权完整的本地启动闭环，并希望尽快获得可直接访问的项目地址。
- 用户偏好先通过多 Agent 详细探索形成明确方案，再授权按完整计划持续实施并以可验证证据收口。
- 用户在大型重构完成后倾向先推送经验证的代码形成稳定节点，再继续外部环境部署。
- 用户会逐项补齐服务器、数据库、对象存储和飞书配置并要求持续推进，期望执行方自主闭环，只在外部权限或资源缺失时汇报。
- 用户使用共享云资源快速首发时，倾向避免会重启实例或波及其他业务的控制台变更，并接受经过实测和显式门禁约束的内网方案。
- 用户会主动指出共享基础设施的潜在爆炸半径，并要求把关键既有业务边界写成可机器验证、失败即停止的发布护栏。
- 用户会从生产截图同时识别品牌资源、登录边界和视觉还原问题，并期望三类问题在同一发布闭环中被真实浏览器验证。
- 用户会在真实使用路径中继续补充登录态体验缺口，并期望修复经全量门禁后直接合并主干、部署同一提交并在线复验。
