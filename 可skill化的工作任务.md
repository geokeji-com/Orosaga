# 可 Skill 化的工作任务

统计窗口：2026-07-16 至 2026-07-22。

| 频次 | 任务抽象名称         | 核心洞察                                                                                           | 关键点                                                                                                          |
| ---: | -------------------- | -------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
|    4 | 本地工具云服务化重构 | 云化不是部署静态站，而是把身份、数据、权限、同步和运维从客户端状态中系统性剥离。                   | 资产与敏感数据盘点；分层单仓；SSO 与 RBAC；私有资产同源代理；外部同步快照；迁移对账；回环灰度；证书切换与回退。 |
|    3 | 云资源权限与联通预检 | 云服务上线前应从真实运行环境逐项验证身份、数据范围、网络路径和最小权限，而不能把配置完成视为可用。 | 应用 token；资源可见范围；私网解析；只读数据库查询；对象存储 ACL；错误码归因；仅输出脱敏计数。                  |
|    2 | 共享宿主机零影响上线 | 新服务上线必须把同机既有域名、证书和路由当作不可变边界，而不能只验证新服务自身。                   | 精确 vhost；独立证书目录；前后哈希；SAN 与内容复核；回环灰度；503 隔离；失败即停止；禁止默认站点回落。          |
|    2 | 本地项目启动与验收   | “启动”任务的完成条件是提供可访问实例，而非仅执行启动命令。                                         | 识别启动入口；检查依赖与端口；不打断既有服务；保持进程；验证首页、入口资源和监听状态。                          |
|    1 | 大型重构提交与推送   | 目录级重构应在远端基线无漂移且发布门禁通过后形成单一、可追溯的交付提交。                           | 同步远端；核对迁移与删除；扫描密钥和产物；运行关键校验；提交；推送；比对本地与远端提交。                        |
