# 项目经验

- 本地启动 Orosaga 时，先补齐 npm 依赖，再用 `npm run dev -- --host 127.0.0.1` 启动，并以根页面和 `/src/main.tsx` 均返回 HTTP 200 且端口持续监听作为完成标准。
- 将本地静态门户云化时，应先把人员资料、外部节点标识和内部地址移出 Web 构建边界，再通过默认拒绝匿名访问的服务端 API 按权限回填。
- 推送大规模目录重构前，应先同步远端、核对删除与迁移关系，并让格式、类型、敏感信息和构建产物门禁全部通过后再提交。
- 一次性人员与头像迁移资料只能由 seed 容器只读挂载，不得进入 API、Worker 或 Web 镜像，并应在生产全量对账成功后从当前分支和服务器 release 移除。
- 国内服务器无法拉取固定基础镜像时，应下载官方运行时归档并校验固定哈希后构建本地镜像，同时独立固定包管理器版本且保留官方镜像作为 CI 默认来源。
- 生产使用 RDS 内网明文连接时，应要求显式 `sslmode=disable` 和独立授权开关，并在写入前确认主机名只解析到私网地址且实际连接未启用 TLS。
